# MiniMax Router

## 核心功能

### 1. 自然语言自动路由（无需命令）

系统自动识别意图并调用对应模型：

| 意图 | 触发关键词 | 示例 |
|------|-----------|------|
| 图片 | 生成图片、画一张、做个图 | "帮我生成一张海边日落图" |
| 视频 | 生成视频、做个视频 | "帮我做一个日出视频" |
| 视频+配音+音乐 | 视频配音、视频旁白、视频音乐 | "生成视频并配音" |
| 音乐 | 写首歌、作曲 | "帮我写首摇滚歌曲" |
| 语音 | 转语音、读出来、配音 | "把这段文字转语音" |
| 对话 | 其他所有 | "今天天气怎么样" |

### 2. 显式斜杠命令

```
/c <内容>   →  文本对话
/t <内容>   →  文本转语音
/v <内容>   →  文本/图片转视频
/m <内容>   →  作曲
/i <内容>   →  文本转图像
```

### 3. 链式组合（自然语言）

用连接词组合多个操作：

```
生成一张海边日落图，然后转语音
        ↓
1. 生成图片 → 2. 把描述转语音
```

### 4. 视频+配音+音乐

一句话搞定完整视频：

```
"生成视频：海边日落，旁白：今天天气真好，配乐：轻快钢琴曲"
        ↓
1. 生成视频
2. 生成 TTS 配音
3. 生成背景音乐
4. FFmpeg 合并为最终视频
```

## 调用规范

图片、视频、音乐、语音合成，参数不全时必须追问清楚，确认后再执行。参数齐全时直接执行。

### 作曲流程（D 方案：智能反推 + 三选一）

**核心原则：** 不堆问询清单，先看用户原话能反推多少，反推不全时**给出 3 套候选风格让用户选**，而不是逐项询问。

#### Step 1：识别意图 + 反推

收到"写首歌/作曲/生成音乐"时，先从用户原话里抓：
- 主题/对象（"AI 时代"、"深夜加班"、"给孩子的歌"）
- 情绪/场景（"治愈"、"上头"、"配视频"）
- 明确风格词（"古风"、"民谣"、"摇滚"、"英文歌"）

**反推出来的信息** → 直接用
**反推不出来的** → 进入 Step 2

#### Step 2：智能反推 3 套候选（核心新增）

不说"请告诉我风格/情绪/BPM"，而是根据已抓到的信息，**主动设计 3 套完整方案**：

```
我：根据你提到的「深夜加班」，给你 3 个方向，你挑一个：

  ① 【都市独白·民谣】
     男声 + 木吉他主导 + 78 BPM + 治愈系
     适合：自弹自唱型账号、个人 IP 主题曲
     副歌设计：低开高走，最后一遍上头

  ② 【深夜电台·流行抒情】
     男声 + 钢琴 + 大提琴 + 75 BPM + 故事感
     适合：视频号/口播、深夜情感类内容
     副歌设计：弦乐推进，金句重复

  ③ 【燃系独立·摇滚】
     男声 + 电吉他 + 鼓组 + 130 BPM + 释放感
     适合：抖音挑战、品牌主题曲
     副歌设计：失真吉他墙，情绪爆炸

  也可以告诉我其他方向，或自己定风格。
```

**如果用户只丢了一句关键词**（如"旋律感强"），就按"旋律感"维度推荐：
- 流行抒情 ⭐⭐⭐⭐⭐（最对路）
- City Pop ⭐⭐⭐⭐⭐
- 民谣 ⭐⭐⭐⭐
- 其他反而不推

#### Step 3：确认人声 / 纯音乐

人声歌曲 vs 纯音乐**只在用户没明确说时问一次**：

```
我：你是要有人声演唱，还是纯音乐（无歌词）？
  ① 有人声（默认）
  ② 纯音乐（用作背景音/Vlog BGM）
```

如果用户已经说"写首歌"、"做主题曲"，默认**人声**，跳过这步。

#### Step 4：歌词处理（重要：默认保护用户歌词）

| 用户行为 | 处理 |
|---------|------|
| 直接贴歌词 | ✅ 用用户的，**不**让 AI 改 |
| 说"无/你帮我想" | 调用 `lyrics_generation` 接口自动写词 |
| 已有主题，要 AI 帮忙写 | 先 `lyrics_generation`，**展示给用户看后再生成音乐** |

**关键规则：** `lyrics_optimizer` 参数**默认关闭**。用户辛苦写的词不能被 AI 偷偷改掉。

#### Step 5：参数确认 + 生成

```
我：准备好了：
  模型：music-2.6
  风格：流行抒情 / 钢琴主导 / 男声 / 78 BPM
  歌词：用户自写，508 字
  输出：<路径>.mp3 (256kbps / 44.1kHz)
  配额：今日已用 0/无限

  回复「生成」执行。
```

#### 歌词模板（必须告知用户格式）

**music-2.6 强依赖结构标签**，没标签会被拒（报"lyrics is required"或乱唱）。告诉用户标准格式：

```
[Intro]
（开头纯音乐或开场白，可选）

[Verse 1]
主歌第一段（叙事，4-8 行）

[Verse 2]
主歌第二段（推进，4-8 行）

[Pre-Chorus]
预副歌（情绪铺垫，2-4 行）

[Chorus]
副歌（核心金句，4-8 行，**全曲最洗脑**）

[Verse 3]
主歌第三段（可选）

[Chorus 2]
副歌重复或变奏

[Bridge]
桥段（转折/对比，2-6 行）

[Outro]
结尾（呼应主题，1-4 行）
```

**完整示例**（用刚才给用户写的那首 AI 时代歌曲）：

```
[Verse 1]
十五年没写代码 双手早就生疏
上个月打开 AI 像是被人从梦里叫醒

[Chorus]
扩大认知的广度
把不同领域连成一张网
```

用户用这个格式写词，生成质量稳定 10 倍。

#### 失败重试策略

| 错误 | 处理 |
|------|------|
| `lyrics is required` | 提示用户加 `[Verse]/[Chorus]` 标签 |
| `usage limit exceeded` | 升级提示（如果有限额） |
| `audio_setting` 不支持 | 默认 44100/256k/mp3，不要改 |
| 生成超时 (>10分钟) | 询问是否继续等待 |
| 输出 < 1MB | 八成是失败，提示重试 |

#### 一句话总结

> **不追问参数清单，给 3 套候选让用户挑；默认保护用户歌词；强制要求结构标签。**

### 图片生成流程

```
用户：我要生成图片
我：请一次性告诉我：
    1. 图片内容/主题
    2. 比例：①1:1 ②16:9 ③9:16 ④4:3 ⑤3:4

用户：海边日落，②
→ 执行
```

参数齐全时（如"帮我画一张海边日落图"）直接执行，不追问。

### 视频生成流程

```
用户：帮我做视频（或直接发来一张图片）
→ 直接执行，默认 768P / 6秒 / Fast模型
```

有图时自动用 Fast 模型；无图时用 2.3 模型。分辨率默认 768P，时长默认 6秒，无需询问。

### 语音合成流程 (MiniMax Speech 2.8)

**完整引导流程（按步骤执行）：**

```
用户：我要文字转语音 / 帮我转语音 / 把这段话转语音 / 转成语音
↓
我：请提供要转换的文字内容
用户：[用户输入文字内容]
↓
我：根据内容分析，智能推荐最合适的5种音色（附推荐理由）

示例输出：
  根据您内容的类型，推荐以下音色：

  ① 电台男主播 — 适合：财经/投资/资讯类内容，播报感强
  ② 沉稳高管 — 适合：商业/金融/分析类内容，权威可信
  ③ 新闻女声 — 适合：新闻/公告/政策类内容，正式清晰
  ④ 温润青年 — 适合：故事/散文/生活类内容，亲切自然
  ⑤ 甜美女声 — 适合：娱乐/轻松/分享类内容，柔和舒适

  请选择音色（①-⑤）或说"音色克隆"使用自己的声音

用户：①
↓
我：请选择情绪表达方式：
    A. 自动（根据内容自动匹配情绪）
    B. 手动选择
用户：B
我：为这段内容推荐以下情绪：
    ① 开心/明亮 — 积极正面的播报
    ② 平静/严肃 — 沉稳专业的表达
    ③ 惊讶/兴奋 — 强调重点和亮点
用户：②
↓
我：自然停顿级别？
    控制语气词（嗯、呃、哎等）的使用频率：

    ① 关闭 — 不用语气词，正式严谨
    ② 轻度 — 仅在段落间/长句停顿处加
    ③ 自然 — 内容自适应，轻松内容加，正式内容少加

    为您的内容推荐：②轻度

用户：[选①/②/③ 或直接回车接受推荐]
↓
→ ⚠️ 语音合成已准备好
  音色：电台男主播
  情绪：平静/严肃
  停顿级别：轻度
  内容：[用户内容摘要...]
  回复「生成」确认执行
用户：生成
→ 执行 TTS，生成音频文件
```

**内容类型 → 音色推荐规则：**

| 内容关键词 | 推荐音色 | 推荐理由 |
|-----------|---------|---------|
| 投资/理财/股票/市场/巴菲特/财报 | 电台男主播 | 财经资讯感强，节奏专业 |
| 商业/企业/管理/职场 | 沉稳高管 | 商务权威感，可信度高 |
| 新闻/公告/政策/官方 | 新闻女声 | 正式清晰，信任感强 |
| 故事/小说/散文/随笔 | 温润青年 | 亲切自然，叙述感好 |
| 娱乐/八卦/轻松/分享 | 甜美女声 | 柔和舒适，轻松愉快 |
| 教育/培训/课程/教程 | 温润青年 | 亲切教学，易于接受 |
| 健康/养生/医疗 | 甜美女声 | 温柔关怀，信任感 |
| 科技/互联网/AI | 电台男主播 | 专业科技感强 |
| 儿童/童话/绘本 | 甜美女声 | 活泼可爱，贴近孩子 |
| 诗歌/文学/经典诵读 | 温润青年 | 文艺气质，节奏感好 |

如果内容涵盖多个类型，选择最突出的一个作为主要推荐理由。

**自然停顿级别 → 内容自适应默认值：**

| 内容类型 | 默认级别 | 说明 |
|---------|---------|------|
| 财经/投资/股票/新闻/公告/政策 | 轻度 | 正式专业，少加语气词 |
| 商业/企业管理/分析报告 | 轻度 | 商务严谨 |
| 故事/散文/随笔/分享 | 自然 | 亲切自然，允许适当停顿 |
| 娱乐/八卦/轻松/闲聊 | 自然 | 轻松愉快 |
| 教育/培训/教程 | 轻度 | 教学专业感 |
| 儿童/童话/绘本 | 自然 | 活泼可爱 |
| 对话/采访/问答 | 自然 | 模拟真人对话节奏 |

用户可以手动选择其他级别覆盖默认值。

**音色克隆（高级功能）：**
```
用户：我要用自己的声音
我：请提供一段参考音频的 URL（10秒以上效果最佳）
    支持 MP3、WAV 格式，建议音频清晰无噪音
用户：https://example.com/my_voice.mp3
→ 继续引导音色选择后的流程
```

**自动情绪检测规则：**
- 文本含"太好了"、"开心"、"大涨"、"涨停" → happy
- 文本含"难过"、"伤心"、"大跌"、"亏损" → sad
- 文本含"愤怒"、"可恶"、"离谱" → angry
- 文本含"震惊"、"震撼"、"重磅" → surprised
- 其他情况默认 → calm（平静）

**语气词使用原则：**
- 轻度：每 100 字不超过 1-2 个语气词，只在段落间自然停顿处
- 自然：每 100 字不超过 3-4 个语气词，段落间 + 长句中适当位置
- 禁用场景：诗歌朗诵、新闻播报（即使选了自然也自动降为轻度）

### 确认机制

参数齐全时，执行前仍需确认：
```
用户：帮我生成一张海边日落图，16:9
→ ⚠️ 图片生成已准备好。
  内容：海边日落 | 比例：16:9
```

### 普通对话
文字对话直接执行，无需确认。

## 模型对应

| 功能 | 模型 | Token Plan 限额 (Max) |
|------|------|----------------------|
| 语音 TTS | speech-2.8-hd | 11,000 字/天 |
| 图片 Image | image-01 | 120 张/天 |
| 视频 Video | MiniMax-Hailuo-2.3 / Fast | 2 个/天 |
| 音乐 Music | **music-2.6** | **无限制**（最新模型，推荐默认） |
| 音乐 Music (旧) | music-2.5 | 4 首/天（仅作 fallback） |

## 视频模型选择（自动判断）

| 你提供的内容 | 调用的模型 | 说明 |
|-------------|-----------|------|
| 只有文字描述 | MiniMax-Hailuo-2.3 | 文字生成视频 |
| 提供了图片 | MiniMax-Hailuo-2.3-Fast | 图生视频，速度更快 |

规则：有图 → Fast；无图 → 2.3。Fast 模型不支持纯文字生成视频。

## 脚本

- `router.py` — 自动路由主脚本
- `tts.py` — 语音合成 (v2.0，支持情绪/克隆/语气词)
- `image.py` — 图像生成
- `video.py` — 视频生成
- `music.py` — 音乐生成（默认 `music-2.6`，支持 `--model / --instrumental / --lyrics-optimizer`）
- `video_with_audio.py` — 视频+配音+音乐合并

### music.py CLI 用法

```bash
# 带歌词（默认模型 music-2.6）
python music.py -p "Mandopop, Emotional, Male vocal" -l "歌词带 [Verse][Chorus] 标签" -o output.mp3

# 纯音乐
python music.py -p "Lo-fi, Study, Calm, 70 BPM" --instrumental -o bgm.mp3

# 显式指定模型（fallback 到 2.5）
python music.py -p "..." -l "..." --model music-2.5 -o out.mp3

# 让 AI 帮你写词（不建议，默认不开启）
python music.py -p "Mandopop, 友情, 毕业" --lyrics-optimizer -o out.mp3
```

### music.py 调用规范

**保护机制：** 默认会拒绝"只传 -p 没歌词也没其他参数"的裸调，避免误用。
**绕过方式：** 带 `-l`（歌词）/ `--instrumental`（纯音乐）/ `--model <非默认值>` 三个之一即可直接调用。
